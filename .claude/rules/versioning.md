# バージョニングルール

tileui は [Semantic Versioning](https://semver.org/spec/v2.0.0.html) に準拠する。

## バージョン判断基準

| 変更内容 | バージョン | 例 |
|---|---|---|
| 破壊的変更（API 削除・変更） | メジャー（0.x の間はマイナー） | 0.1.0 → 0.2.0 |
| 機能追加（後方互換） | マイナー（0.x の間はパッチ） | 0.1.0 → 0.1.1 |
| バグ修正 | パッチ | 0.1.0 → 0.1.1 |
| 依存パッケージのアップデート（runtime deps） | パッチ | 0.1.0 → 0.1.1 |
| devDependencies のみのアップデート | バージョン変更なし | - |
| ドキュメント・CI のみの変更 | バージョン変更なし | - |

## 0.x 期間の特別ルール

- 1.0.0 未満は安定版ではないため、破壊的変更をマイナーで扱う
- 1.0.0 リリース後は通常の Semantic Versioning に移行する

## バージョンアップ手順

1. `package.json` の `version` フィールドを更新
2. `CHANGELOG.md` の `## [Unreleased]` セクションに変更内容を記載済みであることを確認
3. `## [Unreleased]` を `## [x.y.z] - YYYY-MM-DD` に変更し、新しい `## [Unreleased]` セクションを追加
4. コミットメッセージ: `release: vX.Y.Z`
5. タグ作成: `git tag vX.Y.Z`
6. push（タグ push で publish.yml が npm 公開を実行）

## 提案タイミング

- PR 作成時に、ブランチの全コミットを分析してバージョンアップを提案する
- バージョン対象の変更が含まれる場合、PR 作成前にバージョンアップコミットの追加を提案する
- バージョン対象外（ドキュメント・CI のみ）の場合は提案しない

## 注意事項

- バージョンアップは Claude が変更内容から判断して提案し、ユーザーが承認する
- 複数の変更がある場合は、最も大きいバージョン変更に合わせる
- バージョンアップのコミットは変更内容のコミットとは分離する
